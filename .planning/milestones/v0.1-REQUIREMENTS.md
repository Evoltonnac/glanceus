# Archived REQUIREMENTS - Quota Board (v0.1)

All requirements for v0.1 have been implemented and validated.

## A. 配置与集成（Configuration & Integration）

- [x] `REQ-CFG-000` 平台接入策略必须保持平台无关性。
- [x] `REQ-CFG-001` 系统必须自动扫描并加载 `config/integrations/` 下的 Integration YAML 文件。
- [x] `REQ-CFG-002` 每个 Integration 必须支持定义 `id/name/description/flow/templates`。
- [x] `REQ-CFG-003` Flow 必须支持步骤类型：`api_key`、`oauth`、`curl`、`webview`、`http`、`extract`、`script`、`log`。
- [x] `REQ-CFG-004` Flow 变量解析必须遵循优先级：`outputs > context > secrets`。
- [x] `REQ-CFG-005` Flow 必须采用显式安全策略。
- [x] `REQ-CFG-006` 视图模板字段必须支持动态模板求值。
- [x] `REQ-CFG-007` v1 测试基线必须覆盖 API Key + Fetch, cURL/WebView, OAuth。

## B. 执行引擎与交互（Execution & Interaction）

- [x] `REQ-EXE-001` 系统必须支持定时执行采集任务（APScheduler）。
- [x] `REQ-EXE-002` 交互型步骤进入可恢复的暂停状态。
- [x] `REQ-EXE-003` 交互信息补齐后恢复执行。
- [x] `REQ-EXE-004` 单个数据源失败不得阻塞其他。
- [x] `REQ-EXE-005` 数据源运行状态持久化。
- [x] `REQ-EXE-006` 默认采集频率 30 分钟。
- [x] `REQ-EXE-007` `webview`/`curl` 场景默认采集频率 60 分钟。
- [x] `REQ-EXE-008` Step 超时策略按工具类型设定。
