# Phase 1 Plan 02: Rust Daemon Claim Loop and Direct Callback

## Objective
- **What:** Add runtime-level daemon that polls backend tasks and executes scraper worker without UI coupling.
- **Why:** Guarantee scraper starts and completes when UI is minimized/hidden/invisible-to-user.
- **Output:** Tauri claim/heartbeat loop + direct complete/fail callback from scraper runtime.
